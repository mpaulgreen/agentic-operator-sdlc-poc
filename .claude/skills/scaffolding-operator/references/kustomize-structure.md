# Kustomize Configuration Structure

The `config/` directory uses Kustomize overlays to manage Kubernetes manifests. Each subdirectory is a Kustomize base or overlay.

## config/default/

Top-level kustomization that combines all other bases.

**kustomization.yaml**:
- `namespace: {{OPERATOR_NAME}}-system`
- `namePrefix: {{OPERATOR_NAME}}-`
- Resources: `../crd`, `../rbac`, `../manager`
- Contains commented-out sections for webhook, cert-manager, prometheus enablement

**manager_config_patch.yaml**: Kustomize patch for additional manager Deployment configuration.

> **Note**: Previous versions used a `manager_auth_proxy_patch.yaml` to inject a `kube-rbac-proxy` sidecar for secure metrics. This is deprecated since operator-sdk v1.33+. Controller-runtime v0.17+ provides built-in metrics authentication via `filters.WithAuthenticationAndAuthorization` in `main.go`. The manager serves authenticated HTTPS metrics directly on port 8443 — no sidecar needed.

## config/manager/

Controller manager Deployment.

**kustomization.yaml**:
- Resources: `manager.yaml`
- Images: maps `controller` → actual registry image

**manager.yaml** (Namespace + Deployment — TWO resources separated by `---`):
- First resource: `kind: Namespace` with `name: system` (kustomize prefixes to `{{OPERATOR_NAME}}-system`). Without this, `make deploy` fails because namespace-scoped resources are created before the namespace exists.
- Second resource: `kind: Deployment` in namespace `system`
- 1 replica
- Labels: `control-plane: controller-manager`, `app.kubernetes.io/name: {{OPERATOR_NAME}}`
- Container:
  - Command: `["/manager"]`
  - Args: `["--leader-elect"]`
  - Image: `controller:latest` (kustomize replaces)
  - Security: `runAsNonRoot: true`, drop ALL capabilities, no privilege escalation
  - Ports: none exposed (metrics on 8080, health on 8081 are internal)
  - Probes: liveness `/healthz` (delay 15s, period 20s), readiness `/readyz` (delay 5s, period 10s)
  - Resources: requests 10m CPU / 64Mi memory, limits 500m CPU / 128Mi memory
- ServiceAccount: `controller-manager`
- TerminationGracePeriod: 10s

## config/rbac/

RBAC resources for the operator.

**kustomization.yaml** resources:
- `service_account.yaml` — ServiceAccount `controller-manager`
- `role.yaml` — ClusterRole `manager-role` (auto-generated by `make manifests` from Go markers)
- `role_binding.yaml` — ClusterRoleBinding binding manager-role to controller-manager SA
- `leader_election_role.yaml` — Role for lease-based leader election (configmaps, leases, events)
- `leader_election_role_binding.yaml` — RoleBinding for leader election

**service_account.yaml**:
```yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  labels:
    app.kubernetes.io/name: {{OPERATOR_NAME}}
    app.kubernetes.io/managed-by: kustomize
  name: controller-manager
  namespace: system
```

**role.yaml**: Initially empty or minimal. Populated by `make manifests` from `//+kubebuilder:rbac` markers in controller code.

**role_binding.yaml**:
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: manager-rolebinding
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: manager-role
subjects:
- kind: ServiceAccount
  name: controller-manager
  namespace: system
```

**leader_election_role.yaml**: Role granting access to leases, configmaps, events in the operator namespace for leader election.

**metrics_reader_clusterrole.yaml**: ClusterRole `metrics-reader` granting GET on `/metrics` non-resource URL. Used by Prometheus to scrape operator metrics.

**metrics_service.yaml**: Service `controller-manager-metrics-service` exposing port 8443 for HTTPS metrics served directly by the controller-manager (using controller-runtime's built-in auth, no sidecar).

**<kind_lower>_editor_role.yaml**: ClusterRole granting end users full CRUD on the custom resource.

**<kind_lower>_viewer_role.yaml**: ClusterRole granting end users read-only access to the custom resource.

## config/crd/

CRD manifest bases.

**kustomization.yaml**:
- Resources: `bases/<group>.<domain>_<kind_plural>.yaml`
- Contains `kubebuilder:scaffold` markers for additional CRDs
- Commented-out patches for webhook and cert-manager CA injection

**kustomizeconfig.yaml**: Teaches kustomize how to substitute name and namespace references in CRDs for webhook conversion.

The CRD YAML itself is auto-generated by `make manifests` — do not hand-edit.

## config/samples/

Example custom resource YAML for users.

**kustomization.yaml**: Lists the sample CR files.

**Sample CR** (`<group>_<version>_<kind_lower>.yaml`):
```yaml
apiVersion: {{API_GROUP}}.{{DOMAIN}}/{{API_VERSION}}
kind: {{KIND}}
metadata:
  labels:
    app.kubernetes.io/name: {{OPERATOR_NAME}}
    app.kubernetes.io/managed-by: kustomize
  name: {{KIND_LOWER}}-sample
spec:
  # TODO: fill in spec fields
```

## config/prometheus/

ServiceMonitor for Prometheus metrics scraping.

**kustomization.yaml**: Lists `monitor.yaml`.

**monitor.yaml**: ServiceMonitor targeting the controller-manager metrics service on port `https` with bearer token auth.

## config/scorecard/

OLM scorecard test configuration. Required by `make bundle`.

**bases/config.yaml**: Base scorecard Configuration resource with empty test stages.

**kustomization.yaml**: Combines base config with test patches.

**patches/basic.config.yaml**: Adds `basic-check-spec` scorecard test.

**patches/olm.config.yaml**: Adds OLM validation tests (bundle-validation, crds-have-validation, crds-have-resources, spec-descriptors, status-descriptors).

## config/manifests/

OLM CSV base template. Required by `make bundle` to generate the final CSV.

**kustomization.yaml**: References the CSV base, default config, samples, and scorecard.
