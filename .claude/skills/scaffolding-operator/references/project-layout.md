# Standard Operator Project Layout

Every Go-based OpenShift operator follows this directory structure, generated by `operator-sdk init` and `operator-sdk create api`.

## Top-Level Files

| File | Purpose |
|------|---------|
| `PROJECT` | Kubebuilder metadata: domain, repo, resources, layout version. Tracks all APIs. |
| `Makefile` | Build automation with ~50 targets for dev, build, deploy, OLM. |
| `Dockerfile` | Multi-stage container build (Go builder ‚Üí distroless runtime). |
| `go.mod` / `go.sum` | Go module dependencies. Core dep: `sigs.k8s.io/controller-runtime`. |
| `.gitignore` | Excludes binaries, test outputs, IDE files, go.work. |

## Directory Structure

### `cmd/main.go`

Single entrypoint. Sets up:
- Runtime scheme (registers core K8s types + custom CRD types)
- Controller manager (metrics, health probes, leader election, webhook server)
- Each controller's reconciler
- Health/readiness endpoints

Contains `kubebuilder:scaffold` markers for code generation:
- `//+kubebuilder:scaffold:imports` ‚Äî API package imports
- `//+kubebuilder:scaffold:scheme` ‚Äî scheme registrations
- `//+kubebuilder:scaffold:builder` ‚Äî controller setup blocks

### `api/<version>/`

One directory per API version. Contains:
- `groupversion_info.go` ‚Äî SchemeBuilder, GroupVersion var, AddToScheme func
- `doc.go` ‚Äî Package doc with `//+kubebuilder:object:generate=true` and `//+groupName=<group>.<domain>`
- `<kind_lower>_types.go` ‚Äî Spec, Status, root type (with kubebuilder markers), list type
- `zz_generated.deepcopy.go` ‚Äî Auto-generated by `make generate` (do not edit)

### `internal/controller/`

One controller file per Kind. Contains:
- Reconciler struct (embeds client.Client, Scheme, Recorder)
- `Reconcile()` method ‚Äî the main reconciliation loop
- `SetupWithManager()` ‚Äî registers watches (For, Owns, Watches)
- RBAC markers (`//+kubebuilder:rbac:...`)

May split into multiple files for complex operators:
- `<kind>_controller.go` ‚Äî main Reconcile + setup
- `<kind>_reconcilers.go` ‚Äî per-resource reconcile methods
- `<kind>_status.go` ‚Äî status update logic
- `<kind>_conditions.go` ‚Äî condition type constants and helpers
- `<kind>_helpers.go` ‚Äî labels, naming, utilities

### `config/`

Kustomize-based configuration. See `kustomize-structure.md` for full details.

| Subdirectory | Purpose |
|-------------|---------|
| `config/default/` | Top-level kustomization combining all others. Sets namespace and namePrefix. |
| `config/manager/` | Controller manager Deployment manifest. |
| `config/rbac/` | ServiceAccount, ClusterRole, ClusterRoleBinding, leader election role. |
| `config/crd/` | CRD YAML bases (auto-generated by `make manifests`). |
| `config/samples/` | Example CR YAML for users to copy. |
| `config/prometheus/` | ServiceMonitor for Prometheus scraping (optional). |
| `config/scorecard/` | OLM scorecard test configuration (optional). |
| `config/manifests/` | OLM CSV base for `make bundle` (optional). |

### `bundle/` (generated by `make bundle`)

OLM packaging. Not scaffolded initially ‚Äî generated later by the `bundling-operator` skill.

### `hack/`

Optional build/dev scripts. Not all operators use this.

### `test/`

Optional test utilities. Integration and E2E tests often live here.
